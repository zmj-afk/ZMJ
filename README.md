# 共享单车需求预测分析项目
## 项目背景
共享单车作为城市绿色出行的重要方式，其需求受到时间、环境、空间、运营等多维度因素影响。本项目以芝加哥共享单车运营数据与气象数据为基础，构建随机森林回归模型对小时级共享单车骑行需求进行预测，并从四大维度（时间、环境、空间、运营）分析各特征对需求的影响程度，旨在为共享单车运营调度、资源配置提供数据支撑。

## 实现流程
1. 数据加载与预处理
加载共享单车骑行数据（trips_clean.csv）和芝加哥气象数据（芝加哥_真实天气_202411_202510.csv）；
对时间字段进行格式转换，提取date（日期）、hour（小时）等时间维度信息；
处理缺失值，确保数据完整性。
2. 需求指标构建
以 “每小时骑行次数” 作为核心需求指标，通过分组聚合（date + hour）统计小时级骑行需求量。
3. 多维度特征工程
融合骑行数据与气象数据，构建四大维度特征：

| 特征维度 |	具体特征 |
|:---|:---|
| 时间特征 |	hour（小时）、weekday（星期几）、is_weekend（是否周末）、is_rush（是否高峰时段）|
| 环境特征 |	temp_avg（平均温度）、precip_mm（降雨量）、wind_max（最大风速）|
| 空间特征 |	station_count（活跃站点数）|
| 运营特征 |	demand_lag1（前 1 小时需求滞后值）|
5. 模型训练与评估
划分训练集（80%）和测试集（20%），随机种子固定为 42；
构建随机森林回归模型（n_estimators=150），使用全量 CPU 核心加速训练；
采用 R²、RMSE、MAE 作为模型评估指标。
6. 可视化分析
针对特征重要性、时段需求趋势、工作日 / 周末差异、温度 / 降雨对需求的影响等维度开展可视化分析。
7. 数据输出
保存建模数据集（四大维度建模数据集.csv）及各类可视化图表。
## 模型效果
| 评估指标 | 数值 | 评价 |
|:---|---:|:---|
